# User Stories: Slack Bot with Claude Agent SDK

## 凡例
- **ペルソナ**: Taro = 開発エンジニア, Hanako = テックリード
- **INVEST**: Independent, Negotiable, Valuable, Estimable, Small, Testable

---

## Epic 1: メンション受信・応答開始

### US-01: チャンネルでのメンション受信
**As a** 開発エンジニア（Taro）,
**I want to** チャンネルで `@Bot` とメンションしてメッセージを送る,
**So that** AIアシスタントに質問や相談ができる。

**受け入れ基準:**
- [ ] チャンネル内で `@Bot メッセージ` を投稿するとBotがイベントを受信する
- [ ] メンション部分（`<@BOT_ID>`）がメッセージテキストから除去される
- [ ] 除去後のテキストがClaude Agentに渡される

**ペルソナ**: Taro, Hanako
**関連要件**: FR-01

---

### US-02: 新規スレッドでの応答開始
**As a** 開発エンジニア（Taro）,
**I want to** チャンネルでメンションした際に新しいスレッドで応答を受け取る,
**So that** チャンネルの会話が整理され、やり取りが追跡しやすくなる。

**受け入れ基準:**
- [ ] スレッド外でのメンションに対し、新しいスレッドとして応答が投稿される
- [ ] 応答はメンションされたメッセージのスレッドに紐づく

**ペルソナ**: Taro, Hanako
**関連要件**: FR-02

---

### US-03: 「考え中...」仮メッセージの表示
**As a** 開発エンジニア（Taro）,
**I want to** メンション直後に「考え中...」のような仮メッセージが表示される,
**So that** Botが応答を処理中であることがすぐに分かる。

**受け入れ基準:**
- [ ] メンション受信後、スレッドに仮メッセージ（例：「考え中...」）が即座に投稿される
- [ ] 仮メッセージはストリーミング応答開始後に更新される

**ペルソナ**: Taro
**関連要件**: FR-04

---

## Epic 2: ストリーミング応答

### US-04: リアルタイムストリーミング表示
**As a** 開発エンジニア（Taro）,
**I want to** Claudeの応答がリアルタイムにSlackメッセージとして更新される,
**So that** 応答の全文を待たずに内容を確認し始められる。

**受け入れ基準:**
- [ ] Claude Agent SDKの `session.stream()` から受信したテキストが逐次Slackメッセージに反映される
- [ ] `chat.update` APIによりメッセージが段階的に更新される
- [ ] ユーザーがテキストの流れをリアルタイムに確認できる

**ペルソナ**: Taro, Hanako
**関連要件**: FR-04

---

### US-05: Slack APIレートリミット対応
**As a** 開発エンジニア（Taro）,
**I want to** ストリーミング更新がSlack APIのレートリミットに抵触しない,
**So that** 応答が途中で停止したりエラーにならない。

**受け入れ基準:**
- [ ] `chat.update` の呼び出し頻度が適切に制御される（バッファリング / スロットリング）
- [ ] レートリミットに近づいても応答テキストが欠落しない
- [ ] 最終的な応答テキストが完全に表示される

**ペルソナ**: Taro
**関連要件**: FR-04, Constraint（Slack APIレートリミット）

---

### US-06: 応答完了時の最終メッセージ確定
**As a** 開発エンジニア（Taro）,
**I want to** ストリーミング完了後に最終テキストでメッセージが確定される,
**So that** 完全な応答を確認できる。

**受け入れ基準:**
- [ ] ストリーム完了時に最終テキストで `chat.update` が実行される
- [ ] 中間更新で欠落したテキストがあれば最終更新で補完される

**ペルソナ**: Taro, Hanako
**関連要件**: FR-04

---

## Epic 3: スレッド会話管理

### US-07: スレッド内での会話継続
**As a** 開発エンジニア（Taro）,
**I want to** スレッド内で `@Bot` とメンションして追加の質問ができる,
**So that** 前の会話の文脈を踏まえた回答を得られる。

**受け入れ基準:**
- [ ] スレッド内でのメンションに対し、同じスレッド内で応答が返る
- [ ] 既存のClaude Agentセッションが再開され、会話コンテキストが維持される

**ペルソナ**: Taro, Hanako
**関連要件**: FR-02, FR-03

---

### US-08: セッション作成と紐づけ
**As a** 開発エンジニア（Taro）,
**I want to** 新しいスレッドの開始時にClaude Agentセッションが自動作成される,
**So that** 明示的なセッション管理を意識せずに会話を始められる。

**受け入れ基準:**
- [ ] 新規メンション時に `unstable_v2_createSession()` で新規セッションが作成される
- [ ] `stream()` メッセージから `session_id` が抽出・保存される
- [ ] `thread_ts` → `session_id` のマッピングがストアに保存される

**ペルソナ**: Taro
**関連要件**: FR-03, NFR-02

---

### US-09: セッション再開による文脈維持
**As a** 開発エンジニア（Taro）,
**I want to** スレッド内の追加メンションで以前の会話コンテキストが保持される,
**So that** 同じ話題について継続的に深掘りできる。

**受け入れ基準:**
- [ ] スレッド内メンション時に `thread_ts` から `session_id` が検索される
- [ ] `unstable_v2_resumeSession()` で既存セッションが再開される
- [ ] Claudeが以前のやり取りを踏まえた回答を返す

**ペルソナ**: Taro, Hanako
**関連要件**: FR-03, NFR-02

---

### US-10: セッション未発見時のフォールバック
**As a** 開発エンジニア（Taro）,
**I want to** セッションが見つからない場合（プロセス再起動後など）でも新しいセッションが作成される,
**So that** 常にBotと対話できる。

**受け入れ基準:**
- [ ] `thread_ts` に対応する `session_id` が見つからない場合、新規セッションが作成される
- [ ] エラーにならず、ユーザーは通常通り応答を受け取れる

**ペルソナ**: Taro
**関連要件**: FR-03, NFR-02

---

## Epic 4: エラーハンドリング

### US-11: Claude Agent SDKエラーの通知
**As a** 開発エンジニア（Taro）,
**I want to** Claude Agent SDKでエラーが発生した場合にスレッドに詳細なエラー情報が表示される,
**So that** 何が問題かを把握し、対処できる。

**受け入れ基準:**
- [ ] Agent SDKのエラー発生時、該当スレッドにエラーメッセージが投稿される
- [ ] エラーメッセージにはエラーの種類と詳細情報が含まれる
- [ ] エラーログがコンソールにも出力される

**ペルソナ**: Taro, Hanako
**関連要件**: FR-05

---

### US-12: セッション作成・再開エラーの通知
**As a** 開発エンジニア（Taro）,
**I want to** セッション作成や再開に失敗した場合にエラー情報が表示される,
**So that** セッション関連の問題を特定できる。

**受け入れ基準:**
- [ ] `createSession` / `resumeSession` の失敗時、スレッドにエラーが投稿される
- [ ] エラーメッセージに失敗の詳細が含まれる

**ペルソナ**: Taro
**関連要件**: FR-05

---

### US-13: ストリーミング中のエラーハンドリング
**As a** 開発エンジニア（Taro）,
**I want to** ストリーミング応答中にエラーが発生した場合、それまでに受信したテキストが保持されエラーが通知される,
**So that** 部分的な応答を失わず、エラーの発生を認識できる。

**受け入れ基準:**
- [ ] ストリーミング中のエラー発生時、受信済みテキストが保持される
- [ ] エラーメッセージがスレッドに追加投稿される
- [ ] エラーの詳細情報がメッセージに含まれる

**ペルソナ**: Taro
**関連要件**: FR-05, FR-04

---

## Epic 5: 設定・起動

### US-14: 環境変数による設定
**As a** 開発エンジニア（Taro）,
**I want to** `.env` ファイルで必要な設定（Slackトークン、APIキー等）を管理できる,
**So that** 設定変更時にコードを修正する必要がない。

**受け入れ基準:**
- [ ] `SLACK_BOT_TOKEN`, `SLACK_APP_TOKEN`, `SLACK_SIGNING_SECRET`, `ANTHROPIC_API_KEY`, `CLAUDE_MODEL` が環境変数から読み込まれる
- [ ] `CLAUDE_MODEL` にデフォルト値が設定されている
- [ ] 必須環境変数が未設定の場合、起動時にエラーメッセージが表示される

**ペルソナ**: Taro
**関連要件**: NFR-04

---

### US-15: Bunでのアプリケーション起動
**As a** 開発エンジニア（Taro）,
**I want to** `bun run` でBotを起動できる,
**So that** 簡単にローカル開発・テストができる。

**受け入れ基準:**
- [ ] `bun run` または `bun run start` でアプリケーションが起動する
- [ ] 起動時にSocket Mode接続が確立され、コンソールにログが出力される
- [ ] 起動成功のメッセージが表示される

**ペルソナ**: Taro
**関連要件**: NFR-01

---

## Epic 6: 開発環境

### US-16: Dev Containerでの開発環境構築
**As a** 開発エンジニア（Taro）,
**I want to** Dev Containerで開発環境が自動的にセットアップされる,
**So that** 環境構築に時間をかけずにすぐに開発を始められる。

**受け入れ基準:**
- [ ] `.devcontainer/` 設定でUbuntuベースのコンテナが構築される
- [ ] Bun、TypeScript等の必要なツールがプリインストールされている
- [ ] Claude Code（Feature: `anthropics.claude-code`）が利用可能
- [ ] VS Code / Claude Code on the webからコンテナを開いてすぐに開発できる

**ペルソナ**: Taro
**関連要件**: NFR-06

---

## Epic 7: ログ・モニタリング

### US-17: 基本ログ出力
**As a** 開発エンジニア（Taro）,
**I want to** Botの動作状況がコンソールにログ出力される,
**So that** 問題発生時にデバッグできる。

**受け入れ基準:**
- [ ] 起動時にアプリケーション起動ログが出力される
- [ ] `app_mention` イベント受信時にログが出力される
- [ ] エラー発生時に詳細なエラーログが出力される

**ペルソナ**: Taro
**関連要件**: NFR-03
