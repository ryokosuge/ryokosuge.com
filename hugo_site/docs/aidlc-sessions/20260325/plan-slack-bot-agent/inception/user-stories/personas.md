# User Personas: Slack Bot with Claude Agent SDK

## Persona 1: 開発エンジニア（Primary）

| 項目 | 内容 |
|---|---|
| **名前** | 田中 太郎（Taro） |
| **役割** | ソフトウェアエンジニア |
| **技術レベル** | 中〜上級（日常的にコードを書く） |
| **利用環境** | Slack Desktop / Web、開発チームのチャンネル |
| **主な動機** | 開発作業中に素早く技術的な回答を得たい |
| **利用頻度** | 日常的（1日数回〜十数回） |

### 利用シナリオ
- コーディング中に技術的な質問を短文でメンション
- コードレビューやデバッグ時に長い文脈（コードスニペット付き）で相談
- エラーメッセージの解析や解決策の相談
- 設計方針やベストプラクティスの確認

### 期待する体験
- メンション後すぐに応答が始まること（リアルタイムストリーミング）
- スレッド内で会話を継続でき、文脈が保持されること
- エラー発生時は詳細な情報が表示され、原因を把握できること
- Slackから離れずにAIアシスタントを利用できること

### フラストレーション要因
- 応答が遅い、または応答が始まったか分からない
- スレッド内で会話を続けても前の文脈を忘れている
- エラーが起きても何が問題か分からない
- 長いメッセージを送ったのに短い応答しか返ってこない

## Persona 2: テックリード / シニアエンジニア（Secondary）

| 項目 | 内容 |
|---|---|
| **名前** | 佐藤 花子（Hanako） |
| **役割** | テックリード |
| **技術レベル** | 上級（アーキテクチャ設計・レビュー担当） |
| **利用環境** | Slack Desktop、複数チャンネルを横断 |
| **主な動機** | チーム内での技術的議論の補助、設計レビューの壁打ち |
| **利用頻度** | 適宜（週に数回） |

### 利用シナリオ
- アーキテクチャ設計の相談（長い文脈付き）
- チームメンバーの質問スレッドに追加質問を投稿
- 技術選定時の比較・検討をBotに依頼
- コードレビュー時の設計改善提案の相談

### 期待する体験
- 深い文脈を理解した上での的確な回答
- スレッド内でチームメンバーと共にBotと対話できること
- 回答のストリーミング中に他の作業を続けられること

### フラストレーション要因
- 表面的な回答しか返ってこない
- スレッドの文脈を無視した回答
