# NFR Requirements: Slack Bot

## NFR-PERF-01: ストリーミング応答レイテンシ

| 項目 | 要件 |
|---|---|
| **カテゴリ** | Performance |
| **要件** | メンション受信から仮メッセージ投稿までの応答時間を最小化する |
| **目標値** | 仮メッセージ投稿まで1秒以内 |
| **計測方法** | イベント受信タイムスタンプと仮メッセージ投稿タイムスタンプの差分 |

---

## NFR-PERF-02: ストリーミング更新頻度制御

| 項目 | 要件 |
|---|---|
| **カテゴリ** | Performance |
| **要件** | 都度更新（チャンクごとに即時更新）しつつ、Slack APIレートリミットに抵触しないこと |
| **制約** | Slack Web API `chat.update` のレートリミット: Tier 3（約50回/分/ワークスペース） |
| **対処** | レートリミット超過時のバックオフ戦略が必要 |
| **優先度** | リアルタイム体験 > API効率 |

---

## NFR-REL-01: セッションフォールバック

| 項目 | 要件 |
|---|---|
| **カテゴリ** | Reliability |
| **要件** | セッション再開失敗時に新規セッション作成にフォールバックすること |
| **条件** | `resumeSession` が例外をスローした場合 |
| **動作** | `createSession` で新規作成し、SessionStoreを更新 |

---

## NFR-REL-02: エラー耐性

| 項目 | 要件 |
|---|---|
| **カテゴリ** | Reliability |
| **要件** | 個別リクエストのエラーがアプリケーション全体をクラッシュさせないこと |
| **動作** | エラーは該当スレッドに通知し、他のリクエスト処理は継続 |

---

## NFR-REL-03: ストリーミング中断耐性

| 項目 | 要件 |
|---|---|
| **カテゴリ** | Reliability |
| **要件** | ストリーミング中にエラーが発生しても蓄積済みテキストが失われないこと |
| **動作** | 蓄積済みテキストを保持した上でエラーメッセージを追加投稿 |

---

## NFR-SEC-01: 認証情報管理

| 項目 | 要件 |
|---|---|
| **カテゴリ** | Security |
| **要件** | 認証情報（トークン、APIキー）がコードにハードコードされないこと |
| **実装** | 環境変数（`.env`ファイル）経由で管理 |
| **注意** | `.env` ファイルは `.gitignore` に含めること |

---

## NFR-MAINT-01: TypeScript Strict Mode

| 項目 | 要件 |
|---|---|
| **カテゴリ** | Maintainability |
| **要件** | TypeScript strict modeで型安全性を確保すること |
| **設定** | `tsconfig.json` で `strict: true` |

---

## NFR-MAINT-02: モジュール分離

| 項目 | 要件 |
|---|---|
| **カテゴリ** | Maintainability |
| **要件** | 関心の分離を意識し、各コンポーネントが単一責務を持つこと |
| **構造** | Application Designで定義したモジュール構成に従う |

---

## NFR-OPS-01: Dev Container開発環境

| 項目 | 要件 |
|---|---|
| **カテゴリ** | Operability |
| **要件** | Dev Containerで開発環境が即座に利用可能であること |
| **ベースイメージ** | Ubuntu |
| **必須ツール** | Bun（最新版）、TypeScript |
| **Feature** | Claude Code（`anthropics.claude-code`） |

---

## NFR-OPS-02: 起動簡便性

| 項目 | 要件 |
|---|---|
| **カテゴリ** | Operability |
| **要件** | `bun run` でアプリケーションを起動できること |
| **前提** | `.env` ファイルに必要な環境変数が設定済みであること |
