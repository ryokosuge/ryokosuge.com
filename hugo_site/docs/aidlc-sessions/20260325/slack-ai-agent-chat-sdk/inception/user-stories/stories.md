# User Stories

## Feature 1: Slack Bot 基本機能

### US-1.1: メンションによるボット呼び出し
**As a** チームメンバー
**I want to** チャンネルで `@bot` とメンションしてボットを呼び出す
**So that** チーム全員が見える場所でAIに質問できる

**Acceptance Criteria:**
- [ ] チャンネル内で `@bot メッセージ` と投稿するとボットが応答する
- [ ] ボットの応答はメンションと同じスレッドに投稿される
- [ ] メンション無しのメッセージにはボットは反応しない

**Persona**: チームメンバー

---

### US-1.2: スレッド内での継続会話
**As a** チームメンバー
**I want to** スレッド内でボットと複数回やり取りする
**So that** 前の会話の文脈を踏まえた回答を得られる

**Acceptance Criteria:**
- [ ] ボットが応答したスレッド内で追加メッセージを送ると、ボットが文脈を保持して応答する
- [ ] スレッド内ではメンション無しでもボットが応答する（subscribeされたスレッド）
- [ ] 過去のスレッド内メッセージを参照した回答ができる

**Persona**: チームメンバー

---

### US-1.3: ダイレクトメッセージでの対話
**As a** チームメンバー
**I want to** ボットにDMで質問する
**So that** チームに見せたくない質問や個人的な相談ができる

**Acceptance Criteria:**
- [ ] ボットにDMを送ると応答が返る
- [ ] DM内でも継続的な会話ができる
- [ ] DMの内容はチャンネルには公開されない

**Persona**: チームメンバー

---

## Feature 2: AI Agent — シンプルな会話

### US-2.1: 自然言語での質問と回答
**As a** チームメンバー
**I want to** 日本語や英語で自由に質問する
**So that** 技術的な質問や一般的な質問にAIが答えてくれる

**Acceptance Criteria:**
- [ ] 自然言語の質問に対してClaude Agent SDKベースのAIが回答する
- [ ] 回答はSlackのメッセージとして読みやすい形式で返される
- [ ] 応答中はタイピングインジケータが表示される

**Persona**: チームメンバー

---

### US-2.2: コンテキストを保持した会話
**As a** チームメンバー
**I want to** 「さっきの件で」「それについてもう少し教えて」のような文脈依存の質問をする
**So that** 毎回前提を説明し直さなくていい

**Acceptance Criteria:**
- [ ] スレッド内の過去メッセージを参照して回答が生成される
- [ ] 「それ」「さっきの」等の指示語を正しく解釈する
- [ ] スレッドが異なれば別のコンテキストとして扱われる

**Persona**: チームメンバー

---

## Feature 3: AI Agent — ツール利用（Claude Code）

### US-3.1: ファイル検索
**As a** チームメンバー
**I want to** 「〇〇というファイルを探して」とボットに依頼する
**So that** ファイルシステム上のファイルを素早く見つけられる

**Acceptance Criteria:**
- [ ] Claude Codeのネイティブファイル検索機能でファイルを検索する
- [ ] 検索結果（ファイルパス）がSlackメッセージとして返される
- [ ] 複数のマッチがある場合はリストで表示される

**Persona**: チームメンバー

---

### US-3.2: コード読み取り
**As a** チームメンバー
**I want to** 「〇〇のファイルの内容を見せて」「この関数の実装を教えて」とボットに依頼する
**So that** 自分でファイルを開かなくてもコードの内容を確認できる

**Acceptance Criteria:**
- [ ] Claude Codeのファイル読み取り機能で指定ファイルの内容を取得する
- [ ] コードはSlackのコードブロック形式で表示される
- [ ] ファイルが大きい場合は該当部分のみ抽出して表示する

**Persona**: チームメンバー

---

### US-3.3: ツール実行結果の提示
**As a** チームメンバー
**I want to** ツールが実行された結果を分かりやすく受け取る
**So that** AIがどのような調査を行ったか理解できる

**Acceptance Criteria:**
- [ ] ツール実行中であることがユーザーに伝わる（タイピングインジケータ等）
- [ ] ツール実行結果を踏まえた回答が自然言語で提供される
- [ ] 必要に応じてツール実行の過程（検索したファイル等）を簡潔に示す

**Persona**: チームメンバー

---

## Feature 4: AI Agent — マルチエージェント

### US-4.1: タスク種類に応じた自動ルーティング
**As a** チームメンバー
**I want to** 質問の種類を意識せずに質問する
**So that** 適切なエージェント（会話 / コード調査）に自動的にルーティングされて最適な回答を得られる

**Acceptance Criteria:**
- [ ] ユーザーの質問内容を分析し、会話エージェントまたはコード調査エージェントに自動ルーティングされる
- [ ] ルーティングはユーザーに透過的に行われる（ユーザーがエージェントを指定する必要がない）
- [ ] コード関連の質問はClaude Codeツールを活用するエージェントにルーティングされる

**Persona**: チームメンバー

---

## Feature 5: 会話状態管理

### US-5.1: 会話履歴の永続化
**As a** チームメンバー
**I want to** ボットが再起動しても過去の会話を覚えている
**So that** ボットのメンテナンス後も途切れなく会話を続けられる

**Acceptance Criteria:**
- [ ] 会話履歴がRedisに永続化される
- [ ] ボット再起動後もスレッドの過去の会話コンテキストが復元される
- [ ] `@chat-adapter/state-redis` を使用して状態管理される

**Persona**: チームメンバー

---

### US-5.2: スレッド単位のコンテキスト分離
**As a** チームメンバー
**I want to** 別のスレッドでは別のトピックについて質問する
**So that** 異なるトピックの会話が混ざらない

**Acceptance Criteria:**
- [ ] 各スレッドが独立したコンテキストを持つ
- [ ] あるスレッドの会話が別スレッドの回答に影響しない
- [ ] DMも独立したコンテキストとして管理される

**Persona**: チームメンバー

---

## Feature 6: 運用・監視（管理者向け）

### US-6.1: ヘルスチェック
**As a** ボット管理者
**I want to** ボットが正常に稼働しているか確認できるヘルスチェックエンドポイントがある
**So that** 監視ツール（CloudWatch等）からボットの死活監視ができる

**Acceptance Criteria:**
- [ ] HTTP ヘルスチェックエンドポイント（例: `/health`）が提供される
- [ ] ボットプロセスが正常な場合は200、異常時は5xxを返す
- [ ] Slack接続状態・Redis接続状態を含むヘルスステータスを返す

**Persona**: ボット管理者

---

### US-6.2: 構造化ログ出力
**As a** ボット管理者
**I want to** ボットの動作ログが構造化された形式で出力される
**So that** CloudWatch等のログ基盤で検索・アラート設定ができる

**Acceptance Criteria:**
- [ ] ログがJSON形式で標準出力に出力される
- [ ] リクエストID、ユーザーID、チャンネルID等のコンテキスト情報が含まれる
- [ ] エラー発生時にスタックトレースがログに記録される
- [ ] ログレベル（info/warn/error）が適切に設定される

**Persona**: ボット管理者
