## Persona 1: チームメンバー（一般ユーザー）

| 項目 | 内容 |
|------|------|
| **名前** | 田中さん（Team Member） |
| **役割** | ソフトウェアエンジニア / デザイナー / PM等 |
| **技術レベル** | Slackの基本操作は問題なし。AI/LLMの専門知識は不要 |
| **動機** | 日常業務の中で素早く情報を取得したい。コードや技術的な質問をAIに聞きたい |
| **利用シーン** | チャンネルでのメンション、スレッド内での継続質問、DMでの個別相談 |
| **期待** | 自然言語で質問すれば的確な回答が返ってくること。ファイル検索やコード読み取りも自動で行ってほしい |
| **フラストレーション** | 回答が遅い、コンテキストを忘れる、ツール利用の結果が分かりにくい |

## Persona 2: ボット管理者

| 項目 | 内容 |
|------|------|
| **名前** | 鈴木さん（Bot Admin） |
| **役割** | インフラ / DevOps / ボットの運用管理担当 |
| **技術レベル** | AWS、Docker、Slack App設定に精通 |
| **動機** | ボットを安定して運用したい。設定変更やトラブルシューティングを効率的に行いたい |
| **利用シーン** | ボットのデプロイ・設定管理、障害対応、エージェント構成の変更 |
| **期待** | ボットの稼働状態が把握しやすいこと。設定変更が容易であること |
| **フラストレーション** | 障害時の原因特定が困難、設定変更に再デプロイが必要 |
