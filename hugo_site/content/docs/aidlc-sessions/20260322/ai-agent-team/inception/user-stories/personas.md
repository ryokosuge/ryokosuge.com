## P-1: Developer（開発者ユーザー）

| 項目 | 内容 |
|---|---|
| **名前** | Developer |
| **役割** | 個人開発者 / システムの利用者 |
| **目的** | 開発アイデアや要件をSlackに投げるだけで、設計・タスク分解・実装まで自動で進めたい |
| **スキル** | ソフトウェア開発経験あり、Slack/GitHub/Devinを日常的に利用 |
| **行動パターン** | 要件を簡潔に投稿し、Agentの議論を見守りつつ必要に応じて介入・修正する |
| **ペインポイント** | 要件整理・設計・タスク分解を自分で全部やるのは時間がかかる |
| **期待** | Agentチームが自律的に議論し、質の高いPlanを生成してくれること |

## P-2: PM Agent

| 項目 | 内容 |
|---|---|
| **名前** | PM Agent |
| **役割** | プロダクトマネージャー / ミーティングファシリテーター |
| **責務** | 要件の整理・スコープ定義・ミーティング進行管理・Planの最終まとめ |
| **入力** | ユーザーの要件投稿、対象リポジトリの情報 |
| **出力** | 整理された要件、スコープ定義、ミーティング議事録、最終Plan |
| **対話スタイル** | 短いメッセージで要点を整理し、他Agentに議題を振る。ユーザーへの確認・承認依頼を行う |

---

## P-3: Design Agent（設計Agent）

| 項目 | 内容 |
|---|---|
| **名前** | Design Agent |
| **役割** | ソフトウェアアーキテクト |
| **責務** | アーキテクチャ検討・設計方針の提案・技術的な実現可能性の評価 |
| **入力** | PM Agentが整理した要件、既存コードベースの構造 |
| **出力** | 設計方針、アーキテクチャ提案、技術的制約の指摘 |
| **対話スタイル** | 技術的な観点から短いコメントで提案・フィードバックを出す |

---

## P-4: Task Agent（タスク分解Agent）

| 項目 | 内容 |
|---|---|
| **名前** | Task Agent |
| **役割** | タスクプランナー |
| **責務** | 設計をDevinが実装可能なタスク粒度に分解・依存関係整理・仕様書レベルのIssue作成 |
| **入力** | PM Agentの要件、Design Agentの設計方針 |
| **出力** | タスク一覧、依存関係図、GitHub Issue用の詳細仕様書 |
| **対話スタイル** | 分解した結果を簡潔に報告し、粒度や順序について他Agentに確認を求める |
